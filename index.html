Getting Started

Up and running in under a minute.

Setup

  1. Download and installOpen the DMG and drag Intake to Applications.
  2. Grant Accessibility accessIntake needs this for the global hotkey. macOS will prompt you on first launch.
  3. Configure destinationsOpen Settings from the menu bar icon and add your apps.
  4. Capture awayPress ⌘⇧Space to open the panel from anywhere.

FAQ

Common questions about Intake.

Why does Intake need Accessibility permission?

The Accessibility permission is required to register a global keyboard shortcut that works from any app. Intake uses this solely for the capture hotkey — nothing else.

Where is my data stored?

Capture history and settings are stored locally in ~/Library/Application Support/Intake. API keys are stored in the macOS Keychain. Nothing is sent to external servers unless you configure a webhook destination.

Which macOS versions are supported?

Intake requires macOS 13.0 (Ventura) or later. It runs natively on both Apple Silicon and Intel Macs.

How do I report a bug or request a feature?

Open an issue on our GitHub Issues page. We have templates for both bug reports and feature requests.

Is Intake open source?

Intake is not currently open source. This repository hosts releases, documentation, and issue tracking. The source code is in a private repository.

How do auto-updates work?

Intake uses the Sparkle framework to check for updates. When a new version is available, you'll see an update notification. Updates are signed with EdDSA to ensure authenticity.
